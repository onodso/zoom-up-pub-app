# Sales Playbook Patterns & Logic

## 🔑 重要な戦略的洞察 (Strategic Insights) (See [Comparison](./ZOOM_PRODUCT_COMPARISON.md))
- **対Microsoft戦略**: 
    - E3以下: Blue Ocean (ZPで電話インフラを握る)
    - E5: 全庁リプレイス狙わず、特定部署への「局所最適」で **Zoom Phone only or ZP + ZRA** 差し込み

## 📂 用途カテゴリ (Usage Categories)
1.  **働き方改革**（職員・教員）
2.  **窓口DX** (市民サービス、ワンストップ)
3.  **BCP防災** (情報発信、緊急連絡)
4.  **庁内ICT** (インフラ、デジタル市役所)
5.  **遠隔授業** (教育、不登校支援)
6.  **完全デジタル市役所** (奈良市モデル)

## 🧩 提案パターン (Solution Patterns)

### 1. ZP + AI Concierge (Window DX)
- **用途**: 電話自動対応・意図ベースルーティング (SMS除外)
- **ターゲット**: 窓口DX、コールセンター、住民サービス向上
- **キーワード**: シンプル版コールセンター, 自動応答, 窓口, AIボイスエージェント, コンタクトセンター

### 2. ZP + AI Companion (AIC) (Work Style)
- **用途**: 通話の自動要約・文字起こし (電話リプレイス)
- **ターゲット**: 庁内ICT、働き方改革
- **キーワード**: PBX, 電話交換機, 内線, 固定電話, AI議事録, 通話録音, 自動通話録音, 通話要約, 簡易議事録作成

### 3. ZP + AIC + ZRA (High Value Analysis / Kasuhara)
- **用途**: 会話の高度分析、カスハラ対策、リスク検知
- **ターゲット**: 特定部署の品質向上, パターン2の上位互換, 会話の資産化, インサイト取得, PDCA
- **キーワード**: 通話分析, 音声認識, モニタリング, 品質向上, 感情分析, トーク分析, コールセンター分析, カスハラ対策, リスク検知, 会話資産

### 4. ZM + ZR + ZRA (Education / Oita Model)
- **用途**: 教育インテリジェンス (授業の質の可視化)
- **ターゲット**: 教育委員会、GIGAスクール、遠隔授業, 会話の資産化, インサイト取得
- **キーワード**: 学校, GIGA, 教育, 授業, 教委, 遠隔授業, 不登校支援, 大分モデル, 会話の高度分析

### 5. All-in / Integrated (Municipality DX)
- **用途**: オムニチャネル分析、完全なデジタルツイン
- **ターゲット**: 庁内ネットワーク刷新、全庁DX, 会話の資産化
- **キーワード**: 自治体DX, 庁内ネットワーク, グループウェア, スマートシティ, デジタル田園都市, 会話資産のPDCA

### 6. ZM + AIC (Meeting)
- **用途**: Web会議の効率化
- **ターゲット**: 働き方改革, 通話録音, 自動通話録音, 通話要約
- **キーワード**: Web会議, Zoom, Teams, Webex, 簡易議事録作成

### 7. ZCC + ZP + ZVA (Nara City Model)
- **用途**: 完全デジタル市役所 (24/365 AI対応)
- **ターゲット**: 職員の電話業務の圧倒的負荷減少, 窓口DX, 市民サービス向上, オムニチャネル
- **キーワード**: デジタル市役所, 奈良市モデル, ワンストップ, 24時間365日, 自己解決, ZCC, ZVA
